# Single-Monitor-Desktop (AutoHotkey v2)

### 🚀 仅针对当前显示器显示桌面（多屏协同效率工具）

## 📖 项目简介
**Single-Monitor-Desktop** 是一个基于 AutoHotkey v2 开发的轻量级工具。它弥补了 Windows 原生系统在多显示器环境下的一个功能缺陷：**无法局部清空桌面。**

在 Windows 默认逻辑中，按下 `Win + D` 会强制最小化**所有**显示器上的窗口。而本项目允许你仅针对**当前鼠标所在**的显示器执行“显示桌面”操作，保持其他显示器上的工作流不被打断。

---

## 💡 为什么要使用它？（解决痛点）
在多屏办公或娱乐场景中，原生 `Win + D` 往往显得“用力过猛”：

1. **参考对照：** 你在屏幕 A 看着教程、参考资料或监控图表，想清空屏幕 B 开始写代码或绘图。使用原生快捷键会导致 A 屏的参考资料也被隐藏。
2. **演示与会议：** 你正在屏幕 A 进行投屏分享或线上会议，需要临时回到桌面找文件，但不希望屏幕 B 上的私人聊天窗口或笔记被观众看到。
3. **专注模式：** 某个屏幕堆积了太多临时窗口（如文件夹、临时浏览器页），你只想清空这一个屏幕重新排列，而不愿打断另一个屏幕上正在播放的视频或渲染任务。

**本脚本让窗口管理拥有了“边界感”，实现了真正的多任务并行。**



---

## ✨ 核心功能
* **精准识别：** 脚本通过鼠标位置自动判断目标显示器，仅操作该区域内的窗口。
* **开关模式 (Toggle)：** 第一次按下快捷键最小化窗口，再次按下自动恢复，无需手动一个一个拉回。
* **焦点记忆：** 在恢复窗口后，脚本会自动重新激活（Focus）之前处于活跃状态的窗口，实现无缝切换。
* **高性能执行：** 针对 AHK v2 进行了优化，执行延迟极低，观感接近系统原生动画。

## ⚠️ 环境要求
* **操作系统：** Windows 10 / 11
* **版本要求：** [AutoHotkey v2.0+](https://www.autohotkey.com/)
    * *注意：本项目完全基于 AHK v2 语法编写，不兼容旧版 v1.1。*

## 🚀 快速上手

### 1. 下载与运行
1. 克隆本仓库或直接下载 `SingleScreenDesktop.ahk`。
2. 确保已安装 AutoHotkey v2 环境。
3. 双击运行脚本。任务栏托盘出现绿色 **H** 图标即表示已就绪。

### 2. 默认快捷键
* **`Alt + D`**：切换当前屏幕的桌面显示/恢复。
